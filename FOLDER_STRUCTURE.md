# Trendomic Backend – Folder Structure

Modular monolith layout. Each module is self-contained and can be extracted into a microservice later.

## Request flow

1. Client → API endpoint (with optional `Authorization: Bearer <token>`).
2. **ThrottlerGuard** (global) → rate limit.
3. **JwtAuthGuard** (global) → validate JWT; skip if route has `@Public()`.
4. **RolesGuard** (per-route) → enforce `@Roles(ADMIN)` where used.
5. **Controller** → `class-validator` DTOs via `ValidationPipe`.
6. **Service** → Prisma (DB) and/or **PhysicsMathService** for compute.
7. Response → client.

## Extracting a microservice

Example: extract **Physics & Math Engine**:

- Move `src/physics-math/` to a new NestJS app.
- Replace Prisma usage with HTTP client calls to the main API for user/task persistence, or add a minimal DB in the new service.
- Expose the same `POST /compute/*` endpoints; main API can proxy or call this service internally.
